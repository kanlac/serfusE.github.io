去并发，R(esume)BL…

2018-10-27

中间间隔一周，玩一阵歇一阵，恢复正轨~

Pelican 1.0

静态博客基本完成，托管在 GitHub Pages，从这篇开始，系列周记就都放到这个博客里面了，Medium 大概可以弃了吧。GitHub Pages 在墙内速度还挺快，这是我没有预料到的。

暂时先告一段落，之后自己使用的过程中应该能积攒不少想法，下次改进时模版也需要重写。如果有朋友有兴趣体验用静态博客写文章，我甚至可以写一份详细的使用文档。

中间发现 Git 的使用还不太熟练，以及需要掌握更多 Sketch 技巧。

由于大部分只是模版的制作，所以练的主要也是前端的东西，CSS 和 HTML 代码有太多模棱两可的地方，这是我不太喜欢写前端的原因。不过调试方面有浏览器开发者工具，这是很方便的一个地方。

简历导向

应该把 PBL 提高一个维度——RBL，基于简历的学习。站在一个比别人低的起跑线上，只能尽量以最最直观的方式把自己的能力展现出来。一个是写博客，二是做项目。

然而一直以来很少写过技术博客，还是因为自视甚渣吧，教程文之类在我看来也只能称作「笔记」而已，「博客」是更 universal 的、分享自己对技术的见解等比较独特的内容。Pelican 虽然不算流行，但也只是一个 SSG（静态站点生成器）工具，而关于 SSG 的文章是屡见不鲜的，Hexo 也好 Jekyll 也好，都大同小异，所以也没什么好讲。

所以在找到我觉得值得写、且适合我写的内容之前，还是先努力凭借项目经验来为我将来的简历填充字画。

恢复单进程

掂量一番，打算以周期性任务代替并发，减少思维的切换，让大脑保持较长时间的连续思考状态，或许是最好的方式。

下一期，Java 框架，和老师安排的设计模式习题，同时开始找能做的外包。基于 RBL 的考虑，搁置 Coursera 的算法课程，因为即使上完我也不知道下一步该怎样做，又会重复忘掉；存储过程没啥可看；Design+Code 在下一期完成后开始。
